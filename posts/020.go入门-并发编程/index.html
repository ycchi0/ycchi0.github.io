020.Go入门-并发编程

October 18, 2024

Go 语言中的并发是其核心功能之一，它通过轻量级的 goroutine 和强大的通信机制（通道，channels）来支持高效的并发编程。Go 以简洁和高效的方式处理并发，而不是像传统语言那样依赖操作系统级线程。Go 的并发模型受到 CSP（Communicating Sequential Processes）模型的启发，注重通过消息传递来实现不同线程之间的协作。

1. Goroutines（协程）#

Goroutine 是 Go 语言的并发执行单元。它类似于线程，但比线程更轻量。Go 的运行时会自动调度 goroutine，而不是依赖操作系统线程调度。一个 Go 程序可以同时运行成千上万个 goroutine，这得益于它的低开销。

语法：只需要在函数调用前加上 go 关键字即可启动一个新的 goroutine。

示例：

package main

import (
    "fmt"
    "time"
)

func say(s string) {
    for i := 0; i < 5; i++ {
        fmt.Println(s)
        time.Sleep(100 * time.Millisecond)
    }
}

func main() {
    go say("Hello") // 启动一个新的 goroutine
    say("World")    // 在主 goroutine 中运行
}

在这个例子中，say("Hello") 和 say("World") 运行在不同的 goroutine 中。这展示了 Go 的并发特性。

2. 通道（Channels）#

Channels 是 Go 提供的一种通信机制，用于在不同的 goroutine 之间传递数据。可以通过通道将数据从一个 goroutine 发送到另一个 goroutine。Go 鼓励开发者“不要通过共享内存来通信，而是通过通信来共享内存”。

通道是类型安全的，意味着你只能发送和接收特定类型的数据。

创建通道：

ch := make(chan int)

发送和接收数据：

  • 发送数据到通道：ch <- value
  • 从通道接收数据：value := <- ch

示例：

package main

import (
    "fmt"
)

func sum(a []int, c chan int) {
    total := 0
    for _, v := range a {
        total += v
    }
    c <- total // 将计算结果发送到通道
}

func main() {
    a := []int{1, 2, 3, 4, 5}
    c := make(chan int)

    go sum(a[:len(a)/2], c) // 启动 goroutine 计算一半的和
    go sum(a[len(a)/2:], c) // 启动 goroutine 计算另一半的和

    x, y := <-c, <-c // 从通道接收两个结果
    fmt.Println(x, y, x+y)
}

在这个例子中，两个 goroutine 并行计算数组的不同部分，并通过通道将结果发送回主 goroutine。

3. 缓冲通道（Buffered Channels）#

默认情况下，通道是无缓冲的，意味着只有在发送方和接收方都准备好时，数据才能在它们之间传递。如果发送方 goroutine 发送数据而没有接收方准备好接收，发送方会阻塞。

缓冲通道允许你指定可以存储多少条消息。它不会像无缓冲通道那样立即阻塞，除非缓冲已满。

示例：缓冲通道：

package main

import (
    "fmt"
)

func main() {
    ch := make(chan int, 2) // 创建缓冲大小为2的通道

    ch <- 1
    ch <- 2

    fmt.Println(<-ch)
    fmt.Println(<-ch)
}

在这个例子中，缓冲通道允许在没有接收方的情况下发送两条消息。如果缓冲已满，再发送数据时会阻塞。

4. 通道的方向#

通道的方向可以限定为发送通道或接收通道。通过限定通道的方向，可以增强程序的可读性并减少错误。

示例：通道方向：

package main

import "fmt"

// 限定只能发送数据的通道
func sendData(ch chan<- int, data int) {
    ch <- data
}

// 限定只能接收数据的通道
func receiveData(ch <-chan int) {
    fmt.Println(<-ch)
}

func main() {
    ch := make(chan int)

    go sendData(ch, 10)
    receiveData(ch)
}

在这个例子中，sendData 函数只能发送数据到通道，receiveData 函数只能从通道接收数据。这种方式可以帮助我们更好地设计 goroutine 间的通信。

5. select 语句#

Go 中的 select 语句用于监听多个通道的通信操作。当多个通道同时可用时，select 会随机选择一个执行。select 语句是 Go 并发模型的核心功能之一，它让处理多个 goroutine 之间的通信更加灵活。

示例：

package main

import (
    "fmt"
    "time"
)

func main() {
    ch1 := make(chan string)
    ch2 := make(chan string)

    go func() {
        time.Sleep(1 * time.Second)
        ch1 <- "from ch1"
    }()

    go func() {
        time.Sleep(2 * time.Second)
        ch2 <- "from ch2"
    }()

    for i := 0; i < 2; i++ {
        select {
        case msg1 := <-ch1:
            fmt.Println(msg1)
        case msg2 := <-ch2:
            fmt.Println(msg2)
        }
    }
}

在这个例子中，select 监听了 ch1 和 ch2 两个通道，并在有数据到达时处理相应的操作。

6. sync.WaitGroup 等同步原语#

有时我们需要确保多个 goroutine 完成工作后再继续执行程序。Go 提供了 sync.WaitGroup 来实现这种同步机制。

示例：使用 WaitGroup 等待多个 goroutine：

package main

import (
    "fmt"
    "sync"
)

func worker(id int, wg *sync.WaitGroup) {
    defer wg.Done() // 标记 goroutine 完成
    fmt.Printf("Worker %d started\n", id)
}

func main() {
    var wg sync.WaitGroup

    for i := 1; i <= 3; i++ {
        wg.Add(1) // 启动新的 goroutine 前增加计数
        go worker(i, &wg)
    }

    wg.Wait() // 等待所有 goroutine 完成
    fmt.Println("All workers done")
}

sync.WaitGroup 允许我们等待所有 goroutine 执行完毕后再继续执行下一步逻辑。

7. sync.Mutex 互斥锁#

在并发编程中，有时我们需要保证多个 goroutine 同时访问共享资源时的安全性。Go 提供了 sync.Mutex 互斥锁来确保某一时刻只有一个 goroutine 能访问共享资源。

示例：使用 Mutex 保护共享资源：

package main

import (
    "fmt"
    "sync"
)

var (
    counter int
    mu      sync.Mutex
)

func increment(wg *sync.WaitGroup) {
    defer wg.Done()

    mu.Lock()   // 获取锁
    counter++   // 临界区
    mu.Unlock() // 释放锁
}

func main() {
    var wg sync.WaitGroup

    for i := 0; i < 1000; i++ {
        wg.Add(1)
        go increment(&wg)
    }

    wg.Wait()
    fmt.Println("Counter:", counter)
}

在这个示例中，使用 Mutex 确保多个 goroutine 同时对 counter 变量进行操作时不会引发数据竞争。

8. 数据竞争与 go test -race#

在并发编程中，数据竞争是指两个或多个 goroutine 同时访问共享数据且至少有一个 goroutine 在修改数据的情况。数据竞争会导致不可预期的错误和行为。

Go 提供了内置的数据竞争检测工具，可以通过 go test -race 来检测代码中的数据竞争问题。

示例：检测数据竞争：

go run -race main.go

如果代码中存在数据竞争问题，-race 选项将会报告这些问题。

9. 并发与并行#

9.1 并发与并行的区别#

在 Go 语言中，并发和并行是两个不同的概念：

  • 并发：并发是指程序可以同时处理多个任务，但不要求这些任务同时执行。并发更注重的是任务的切换和管理，多个任务可能在同一时刻交替执行，使用同一个处理器资源。
  • 并行：并行是指程序可以在多个 CPU 核心上同时执行多个任务。并行是物理上的同时执行，只有在多核处理器上才能真正实现。

Go 通过轻量级的 goroutine 来实现并发，而并行则取决于你运行程序的硬件和配置。Go 语言通过运行时的调度器自动管理并发和并行任务。

9.2 GOMAXPROCS#

Go 提供了 GOMAXPROCS 函数来控制并行的 goroutine 的数量，它决定了程序中可以使用的逻辑 CPU 的最大数量。runtime.GOMAXPROCS 默认值是系统 CPU 核心数，开发者可以根据需要手动调整。

例如，通过以下代码可以设置并行使用的 CPU 核心数：

package main

import (
    "fmt"
    "runtime"
)

func main() {
    fmt.Println("默认的 GOMAXPROCS:", runtime.GOMAXPROCS(0)) // 查看默认值
    runtime.GOMAXPROCS(2) // 设置为使用 2 个逻辑 CPU 核心
    fmt.Println("更改后的 GOMAXPROCS:", runtime.GOMAXPROCS(0))
}

10. Goroutine 调度器#

Go 运行时包含一个高效的调度器，用于管理 goroutine 的调度和执行。Go 语言采用了一种混合的调度模型，结合了抢占式调度和协作式调度，即 goroutine 自己管理让出 CPU 机会，同时运行时也可以在必要时抢占 goroutine 的执行。

调度器通过 P（Processor）、M（Machine）、G（Goroutine）的模型来调度 goroutine：

  • P：表示逻辑处理器，GOMAXPROCS 设置的就是 P 的数量。
  • M：表示操作系统的线程，负责在物理处理器上执行 P 的任务。
  • G：表示 goroutine，是 Go 的并发执行单元。

这种设计使得 Go 能高效地调度成千上万的 goroutine，同时避免了创建过多系统线程的开销。

11. Go 中的并发模式#

Go 语言通过 goroutine 和通道支持多种并发编程模式，以下是几种常见的并发设计模式。

11.1 工作池（Worker Pool）#

工作池是一个常见的并发设计模式，特别适用于需要并行处理大量相似任务的场景。工作池的核心思想是创建一组 goroutine 来处理任务，从而限制并发数量，避免过度创建 goroutine 导致系统资源耗尽。

示例：工作池模式：

package main

import (
    "fmt"
    "sync"
)

func worker(id int, jobs <-chan int, results chan<- int, wg *sync.WaitGroup) {
    defer wg.Done()
    for j := range jobs {
        fmt.Printf("Worker %d processing job %d\n", id, j)
        results <- j * 2 // 模拟任务处理
    }
}

func main() {
    jobs := make(chan int, 100)
    results := make(chan int, 100)

    var wg sync.WaitGroup

    // 启动 3 个 worker
    for w := 1; w <= 3; w++ {
        wg.Add(1)
        go worker(w, jobs, results, &wg)
    }

    // 提交 5 个任务
    for j := 1; j <= 5; j++ {
        jobs <- j
    }
    close(jobs) // 所有任务提交完毕

    // 等待所有 worker 完成
    wg.Wait()
    close(results)

    // 收集结果
    for result := range results {
        fmt.Println("Result:", result)
    }
}

在这个例子中，3 个 worker 同时从 jobs 通道获取任务并处理，并将处理结果发送回 results 通道。

11.2 扇出扇入（Fan-out, Fan-in）#

**扇出（Fan-out）**模式是指将多个任务分发给多个 goroutine 并行处理。**扇入（Fan-in）**模式则是将多个 goroutine 的结果汇总。这个模式适用于并行处理任务，并最终整合结果的场景。

示例：扇出扇入模式：

package main

import (
    "fmt"
    "sync"
)

func worker(id int, jobs <-chan int, results chan<- int, wg *sync.WaitGroup) {
    defer wg.Done()
    for j := range jobs {
        results <- j * j // 模拟计算
    }
}

func main() {
    jobs := make(chan int, 10)
    results := make(chan int, 10)

    var wg sync.WaitGroup

    // 启动多个 worker
    for i := 1; i <= 3; i++ {
        wg.Add(1)
        go worker(i, jobs, results, &wg)
    }

    // 提交任务
    for j := 1; j <= 9; j++ {
        jobs <- j
    }
    close(jobs)

    // 等待所有 worker 完成
    wg.Wait()
    close(results)

    // 收集结果
    for result := range results {
        fmt.Println("Result:", result)
    }
}

这个例子展示了通过多个 goroutine 并发处理任务，并在最后收集所有结果的实现方式。

12. select 多路复用#

Go 语言的 select 语句用于在多个通道上等待数据到达，并执行相应的操作。select 使得程序可以同时监听多个通道，处理不同通道的输入或输出操作。

示例：使用 select 实现多路复用：

package main

import (
    "fmt"
    "time"
)

func main() {
    ch1 := make(chan string)
    ch2 := make(chan string)

    go func() {
        time.Sleep(2 * time.Second)
        ch1 <- "Message from channel 1"
    }()

    go func() {
        time.Sleep(1 * time.Second)
        ch2 <- "Message from channel 2"
    }()

    for i := 0; i < 2; i++ {
        select {
        case msg1 := <-ch1:
            fmt.Println(msg1)
        case msg2 := <-ch2:
            fmt.Println(msg2)
        }
    }
}

在这个例子中，select 同时监听两个通道，并根据哪个通道先发送数据来处理不同的情况。这种方式使得 Go 语言非常适合编写网络服务器或处理多个 I/O 事件的场景。

13. 竞争条件与同步#

在并发程序中，多个 goroutine 可能会同时访问共享数据，从而引发数据竞争问题。Go 提供了多种机制来处理竞争条件并确保 goroutine 的同步。

13.1 sync.WaitGroup#

WaitGroup 是 Go 提供的用于等待一组 goroutine 完成的同步原语。它的典型用法是：每个 goroutine 执行开始时调用 Add 方法，执行完毕后调用 Done 方法，主 goroutine 通过 Wait 方法等待所有 goroutine 完成。

示例：使用 sync.WaitGroup：

package main

import (
    "fmt"
    "sync"
)

func worker(id int, wg *sync.WaitGroup) {
    defer wg.Done() // 标记任务完成
    fmt.Printf("Worker %d starting\n", id)
}

func main() {
    var wg sync.WaitGroup

    for i := 1; i <= 3; i++ {
        wg.Add(1) // 增加计数
        go worker(i, &wg)
    }

    wg.Wait() // 等待所有 worker 完成
    fmt.Println("All workers done")
}

在这个例子中，sync.WaitGroup 确保主 goroutine 等待所有 worker goroutine 完成任务后才继续执行。

015.Go入门-面向对象

October 18, 2024

Go 语言虽然不完全符合传统的面向对象编程（OOP）模型，但它具有一些关键的 OOP 特性，例如封装、组合和多态，这些可以通过 Go 的**结构体（struct）和接口（interface）**来实现。

在 Go 语言中，面向对象的概念不是通过继承和类来实现，而是通过组合与接口设计来模拟和实现多态性和扩展性。

1. Go 语言的结构（struct）#

Go 中的结构体（struct）类似于其他语言中的类，但没有继承关系。struct 是一种聚合数据类型，用于将多个字段组织在一起。与类不同的是，Go 没有构造函数或析构函数的概念，也没有隐式的继承。

定义结构体：

type Person struct {
    Name string
    Age  int
}

使用结构体：

func main() {
    p := Person{Name: "Alice", Age: 25}
    fmt.Println(p.Name)  // 输出：Alice
    fmt.Println(p.Age)   // 输出：25
}

为结构体定义方法#

Go 中的方法是绑定到类型（包括结构体）的。方法的接收者可以是值类型或指针类型，这取决于你想在方法中修改结构体还是只读取它的内容。

type Person struct {
    Name string
    Age  int
}

// 为结构体定义方法
func (p Person) Greet() {
    fmt.Println("Hello, my name is", p.Name)
}

func main() {
    p := Person{Name: "Alice", Age: 25}
    p.Greet()  // 输出：Hello, my name is Alice
}

值接收者 vs. 指针接收者#

  • 值接收者：方法接收者是结构体的一个副本，修改接收者不会影响原结构体。
  • 指针接收者：可以修改接收者本身，通常用于需要修改结构体字段或避免大结构体的副本开销的场景。
func (p *Person) UpdateAge(newAge int) {
    p.Age = newAge  // 直接修改原始对象
}

2. Go 语言中的面向对象#

Go 并没有使用传统 OOP 语言中的类和继承，而是通过组合和接口来实现面向对象的设计模式。

组合（Composition）#

Go 中没有继承，主要依赖组合来复用代码。通过将结构体嵌入到其他结构体中，可以实现类似继承的效果，这种方式被称为“组合”。

type Address struct {
    City    string
    Country string
}

type Person struct {
    Name    string
    Age     int
    Address  // 组合 Address 结构体
}

func main() {
    p := Person{Name: "Alice", Age: 25, Address: Address{City: "New York", Country: "USA"}}
    fmt.Println(p.City)   // 通过组合直接访问嵌套结构体的字段，输出：New York
}

通过组合，你可以让一个结构体包含其他结构体，实现复用已有结构体的属性和方法。

接口（Interface）和多态#

接口是 Go 中实现多态的核心机制。接口定义了一组方法，而任何实现了这些方法的类型都隐式地实现了该接口。与传统 OOP 语言不同，Go 不需要显式地声明一个类型实现了某个接口，Go 的接口是通过**鸭子类型（duck typing）**实现的。

定义接口：

type Speaker interface {
    Speak()
}

实现接口：

type Person struct {
    Name string
}

func (p Person) Speak() {
    fmt.Println(p.Name, "says hello")
}

type Dog struct {
    Name string
}

func (d Dog) Speak() {
    fmt.Println(d.Name, "barks")
}

func main() {
    var s Speaker
    p := Person{Name: "Alice"}
    d := Dog{Name: "Rex"}

    s = p  // Person 实现了 Speaker 接口
    s.Speak()  // 输出：Alice says hello

    s = d  // Dog 实现了 Speaker 接口
    s.Speak()  // 输出：Rex barks
}

在上面的例子中，Person 和 Dog 都实现了 Speaker 接口，因此它们可以通过 Speaker 类型的变量进行多态处理。

3. 封装#

Go 通过大写和小写字母区分可见性。结构体和函数如果以大写字母开头，则是导出的，可以被其他包访问；如果以小写字母开头，则是私有的，只能在定义它的包内使用。

type Person struct {
    Name string  // 公共字段
    age  int     // 私有字段
}

func (p Person) GetAge() int {
    return p.age  // 可以通过公有方法访问私有字段
}

4. 面向对象总结#

  • 结构体：Go 中没有类，结构体（struct）用于组织和存储数据。
  • 方法：可以为结构体定义方法，这些方法可以绑定到值或指针类型的接收者。
  • 组合：Go 中没有继承，通过组合的方式实现代码复用。
  • 接口：Go 中通过接口实现多态，不需要显式声明类型实现了某个接口，只要满足接口方法签名即可。
  • 封装：通过首字母大小写控制字段和方法的可见性。

Go 的面向对象设计更加简洁灵活，它摒弃了传统 OOP 中复杂的继承体系，转而通过组合和接口实现代码复用和多态性，更加贴近简单、高效的编程思想。
